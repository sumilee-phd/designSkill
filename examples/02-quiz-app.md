## 화면별 스펙

### 시작 화면
- Hero: "오늘의 퀴즈를 시작할 준비 되었나요?"
- 서브: "5문항 · 예상 소요 3분 · 각 문항 30초"
- 큰 Play 버튼 (바이올렛 그라디언트, 라운드 full)
- 하단: "문제당 30초 카운트다운이 있어요"

### 문항 화면
- 상단: 진행 바(1/5 · 2/5 ...) + 우측 원형 타이머
- 중앙: 질문 텍스트 28–36px Bold
- 답 4개 버튼 2×2 그리드 (데스크톱) / 세로 4열 (모바일)
  - A=빨강 · B=파랑 · C=노랑 · D=초록
  - 눌렀을 때 즉시 판정:
    - 정답: 초록 글로우 + scale 1.05 + 정답 ⚡ 아이콘 · 점수 +20
    - 오답: shake 애니메이션 + 빨강 배경 · 정답 버튼 초록으로 점등
- 판정 후 0.8초 뒤 "다음 문항 ›" 버튼 표시

### 결과 화면
- 큰 점수 100점 만점 (예: 80점) · 트로피 아이콘 급수 결정
  - 100점: 🏆 명인
  - 80~99: ⭐ 문화유산 박사
  - 60~79: 📘 성실한 학생
  - ~59: 🌱 다시 한 번!
- 문항별 정답/오답 리스트 (해설 보기 토글)
- CTA: "다시 도전" · "결과 이미지 저장" · "친구에게 공유"

## 기술
- 단일 `index.html` · Tailwind CDN · Pretendard · Iconify Solar
- JS는 바닐라(React 없이) — 학교 환경 배포 용이
- 상태: 현재 문항 번호 · 선택 답 · 점수만 추적
- 타이머: `setInterval`로 30초 카운트, 0이 되면 자동 오답 처리
- LocalStorage: 최고 점수 기록

## 접근성
- 답 버튼 `aria-label="A번 서울 종로구"`
- 키보드: 1/2/3/4 숫자키로 답 선택 가능
- 모션 최소 모드 존중 (`prefers-reduced-motion`)
